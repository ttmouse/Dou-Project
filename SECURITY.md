# 安全政策

## 支持的版本

我们仅对以下版本提供安全更新支持：

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | ✅ 支持 |
| < 1.0 | ❌ 不支持 |

## 报告安全漏洞

我们严肃对待 ProjectManager 的安全问题。如果您发现了安全漏洞，请遵循负责任的披露原则。

### 如何报告

**请勿在公共 Issue 中报告安全漏洞。**

相反，请通过以下方式之一私下报告：

1. **通过 GitHub Security Advisories**（推荐）
   - 访问我们的 [Security Advisories](https://github.com/your-username/project-list/security/advisories)
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **通过邮件**
   - 发送邮件至：security@yourproject.com
   - 在邮件主题中包含 "[SECURITY]" 前缀
   - 使用 PGP 加密（密钥：[链接到公钥]）

### 报告内容

请在您的报告中包含以下信息：

- 漏洞的详细描述
- 重现步骤
- 受影响的版本
- 潜在的影响程度
- 任何可能的缓解措施
- 您的联系信息（用于跟进）

### 响应时间

- **初始响应**：48 小时内确认收到报告
- **详细响应**：7 天内提供初步评估
- **修复时间**：根据严重程度，30-90 天内发布修复

## 安全最佳实践

### 用户须知

作为 ProjectManager 的用户，您可以采取以下措施保护自己：

1. **保持更新**
   - 始终使用最新版本的 ProjectManager
   - 定期检查并安装安全更新

2. **权限管理**
   - 仅授予应用必要的文件系统权限
   - 定期审查应用的权限设置

3. **数据保护**
   - 定期备份项目标签数据
   - 避免在项目名称中包含敏感信息

### 开发者须知

如果您是贡献者，请遵循以下安全实践：

1. **代码审查**
   - 所有代码变更都需要经过审查
   - 特别关注涉及文件系统和标签系统的代码

2. **依赖管理**
   - 使用最新的、安全的依赖版本
   - 定期审查和更新第三方依赖

3. **敏感数据**
   - 永不在代码中硬编码密钥、密码或其他敏感信息
   - 使用环境变量或安全的配置管理

## 已知安全考虑

### 标签系统

ProjectManager 直接操作 macOS 文件系统的元数据：

- **风险**：不当的标签操作可能导致数据丢失
- **缓解**：实施原子操作和错误处理
- **建议**：定期备份标签数据

### 文件系统访问

应用需要读取和监控文件系统：

- **风险**：潜在的隐私和安全问题
- **缓解**：仅访问用户显式授权的目录
- **建议**：用户应仔细选择监控目录

### 第三方编辑器集成

应用会启动外部编辑器：

- **风险**：恶意编辑器可能造成危害
- **缓解**：验证编辑器路径和完整性
- **建议**：仅使用可信任的编辑器

## 安全更新流程

1. **评估**：确定漏洞的严重程度和影响范围
2. **开发**：在私有分支中开发修复方案
3. **测试**：全面测试修复方案
4. **协调**：与报告者协调披露时间
5. **发布**：发布安全更新和公告
6. **通知**：通知用户和社区
